# 演習ファイルの実行方法

`exercises/` フォルダの `.cs` ファイルは、Unity に依存しない純粋な C# コードです。
コンソールアプリとして実行することで、Unity を起動せずに動作確認できます。

このドキュメントでは以下の2通りの実行方法を説明します。
状況に応じて使い分けてください。

| 方法 | 向いている場面 |
|---|---|
| **方法A：Visual Studio** | IDE の補完・デバッグ機能を活用したい場合 |
| **方法B：dotnet CLI** | 手軽に実行確認したい場合・IDE を使わない場合 |

---

## 方法A：Visual Studio でコンソールアプリとして実行

### A-1. 新しいコンソールアプリプロジェクトを作成する

1. Visual Studio を起動します
2. スタート画面で「新しいプロジェクトの作成」をクリックします
3. テンプレート一覧から「コンソールアプリ」を選択します
   - 言語が **C#** のものを選んでください
   - **.NET Framework** ではなく **.NET**（または .NET Core）のものを選んでください
4. プロジェクト名を入力します（例：`CsharpExercise`）
5. 保存先を任意の場所に設定します
6. 「作成」ボタンをクリックします

### A-2. exercises/ のファイルをプロジェクトに追加する

1. Visual Studio 右側の「ソリューションエクスプローラー」でプロジェクト名を右クリックします
2. 「追加」→「既存の項目」をクリックします
3. `exercises/` フォルダ内の実行したい `.cs` ファイル（例：`01_exercise.cs`）を選択して「追加」をクリックします
4. プロジェクト作成時に自動生成される `Program.cs` との競合を避けるため、
   以下のいずれかの対処を行ってください：
   - `Program.cs` をソリューションエクスプローラーで右クリック →「プロジェクトから除外」を選択する
   - または `Program.cs` の中身を全て削除する

> 1つのプロジェクト内に `Main` メソッドを持つクラスが複数あるとエラーになります。
> 演習ファイルを1つずつ試す場合は、その都度「プロジェクトから除外」で切り替えてください。

### A-3. 実行する

- **F5**（デバッグ実行）：ブレークポイントで停止できます。エラー発生時に詳細情報が表示されます。開発中はこちらを使います
- **Ctrl+F5**（通常実行）：デバッガーを起動せずに実行します。動作確認だけしたい場合に使います

**実行後にコンソールウィンドウがすぐ閉じる場合：**

`Main` メソッドの末尾に以下を追加すると、キーを押すまでウィンドウが閉じません。

```csharp
Console.ReadKey();
```

### A-4. 実行結果を確認する

- `Console.WriteLine` の出力は画面に表示される「コンソールウィンドウ」に表示されます
- Visual Studio の下部にある「出力」ウィンドウはビルドのログを表示するもので、
  `Console.WriteLine` の結果はここには表示されません

## 方法B：dotnet CLI で実行

### B-1. dotnet CLI とは

dotnet CLI は .NET SDK に含まれるコマンドラインツールです。
Visual Studio をインストールしていれば、多くの場合 `dotnet` コマンドも同時に使える状態になっています。

まず以下のコマンドでインストール済みか確認してください。

```bash
dotnet --version
```

バージョン番号（例：`8.0.100`）が表示されればインストール済みです。
表示されない場合は [方法B-1のハマりどころ](#よくあるハマりどころ) を参照してください。

### B-2. コンソールアプリを作成して実行する

**1. 新しいコンソールアプリを作成します**

```bash
dotnet new console -n CsharpExercise
cd CsharpExercise
```

**2. exercises/ の .cs ファイルをコピーします**

```bash
# Windows の場合
copy ..\exercises\01_exercise.cs .\

# macOS / Linux の場合
cp ../exercises/01_exercise.cs ./
```

**3. 自動生成された `Program.cs` を削除または中身を空にします**

```bash
# Windows の場合（ファイルを削除する）
del Program.cs

# macOS / Linux の場合
rm Program.cs
```

**4. 実行します**

```bash
dotnet run
```

`Console.WriteLine` の出力がターミナルに表示されます。

## よくあるハマりどころ

### dotnet コマンドが見つからない

`dotnet` コマンドが見つからない場合、.NET SDK が別途インストールされていない可能性があります。
[.NET SDK 公式ダウンロードページ](https://dotnet.microsoft.com/ja-jp/download) からインストールしてください。

### コンソールウィンドウがすぐ閉じる

Visual Studio でデバッグ実行（F5）した際にウィンドウがすぐ閉じる場合は、
`Main` メソッドの末尾に以下を追加してください。

```csharp
Console.ReadKey(); // 何かキーを押すまで待機する
```

### Main メソッドが重複してエラーになる

1つのプロジェクトに `Main` メソッドを持つクラスが複数あると、
以下のようなエラーが発生します。

```
エラー CS0017：プログラム 'CsharpExercise' に複数のエントリポイントが定義されています
```

演習ファイルは1ファイルずつ試すようにしてください。
Visual Studio の場合は不要なファイルをソリューションエクスプローラーで右クリック →「プロジェクトから除外」で対処できます。

### Unity 用のコードを実行しようとした場合

`UnityEngine` を参照しているコードをコンソールアプリとして実行しようとすると、
以下のようなエラーが発生します。

```
エラー CS0246：型または名前空間の名前 'UnityEngine' が見つかりませんでした
```

`exercises/` フォルダのコードは Unity に依存せずに書かれているため、このエラーは発生しません。
`02_unity/` フォルダのコードを実行したい場合は、Unity エディタ上で実行してください。
